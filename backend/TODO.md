### 마이그레이션 요구사항
1. DB 초기화 시점마다 데이터가 삽입되어야 함. 그러므로 재활용 가능한 도구를 만들고 싶음.
2. SQL OR 파이썬 파일이어야 함. alembic or sqlalchemy or 스크립트로 입력
3. 구조는 alembic으로 넣음. 그러므로 데이터만 삽입할 수 있도록 해야함.

### 데이터 마이그레이션 흐름
1. 태그 정보 - tag_code, aliases, tag_problem_count (tags.json 활용)
2. level, 제외 여부, 해당하는 문제의 최소 티어, 최대 티어, 최소 푼사람 수(1000명 default), 태그 숙련도 관련(정해둔 기준에 따라 분류)
3. 문제 정보 - problems.json에서 필드에 맞는 정보 삽입
4. 목표 메타 정보 - DAILY, CT, BEGINNER str로 삽입
5. 문제 추천 난이도 필터
    - 필터링
    - 태그 filtering: 선택한 태그의 문제추천
    - 티어 filtering: 선택한 티어보다 낮거나 높은 문제추천 
    - 난이도 filtering: easy/normal/hard/extreme
        ENUM : EASY, NORMAL, HARD, EXTREME
        
        > 해당 태그 문제 중 내가 푼 highest 티어
        내 현재 티어
        > 
        - easy: 내 티어 -5 이하 문제들
        - nomal:
            - 사용자가 해당 태그 Imediate상태 : 50%이하
            - 사용자가 해당 태그 Advanced상태 : 20% 이하
        - hard (이걸풀어야 highest 문제기준):
            - 사용자가 해당 태그 Imediate상태 : 50% - 30%
            - 사용자가 해당 태그 Advanced상태 : 0% - 20%
        - extreme: highest +2 | 내티어 + 2 를 둘다 만족 하는 문제들
        
6. 태그 관계 (부모 태그)
    - 이건 나중에 리스트 줄게.
7. 문제-태그 매핑
    - 태그와 문제의 다대다 관계 삽입
8. 목표-태그 매핑

### 태그 규칙
- 고려대상 문제: “1000명 이상이 푼 문제” -> **데이터 삽입 시 기본값 1000 으로 삽입**

- 쓰레기 태그 여부: **커스텀 (지협적인 태그 | 포괄적태그)**
- 문제별 목표 대상: **(하위에 태그 5단계 참고)** -> **태그의 level. 해당 5단계는 현재  (NEWBIE, BEGGINER, REQUIREMENT, DETAIL,  CHALLENGE) 로 설정됨**

#### 이하는 태그 숙련도의 달성 기준(문제수 기준은 그 아래쪽에서 서술) 해당하는 내용임.
- 추천 시작 기준: 상위태그 전부 Advanced달성 + 최소티어 달성
- 문제 추천 주기: 시작단계(3일) | Advanced(7일) | Master(14일)
- 상위태그: **커스텀**
- ~~하위태그: 개발용~~
- 최소티어: 대상문제 중 상위 90%
- Master
    - Solved: 대상 문제 (기준보다 문제수가 적다면, 90%)
    - user-tier: 대상문제 중 상위 50%
    - highest: 대상문제 중 상위 10%
- Advanced
    - Solved: 대상 문제 (기준보다 문제수가 적다면, 70%)
    - user-tier: 대상문제 중 상위 70%
    - highest: 대상문제 중 상위 40%
- Imediate
    - 추천받기 시작한 단계를 뜻함
    
#### 이하는 태그의 레벨과 목표, 숙련도별 달성해야 할 문제수, 부모태그(괄호 친 부분) 에 해당하는 내용임.
- 태그를 5단계로 구분하는게 어떤가?
    - 쌩기초 ⇒ Beginer, Daily + (15문제/10문제)
        - 수학, 정렬, 기하학, 사칙연산, 정수론(), 조합론, 소수 판정
    - 기초알고리즘 ⇒ Beginer, CT, Daily + (15문제/10문제)
        - 구현(), 그리디 알고리즘(), 문자열(), 브루드포스 알고리즘(), 트리(), 집합과 맵(), 시뮬레이션(구현), 누적합()
        - 너비우선탐색(), 깊이우선탐색(), 해시를 사용한 집합과 맵(집합과 맵), 백트래킹(), 스택(), 재귀(), “임의 정밀도 / 큰 수 연산”()
        - 에라토스테네스의체(), 덱(), 큐()
    - 필수알고리즘 ⇒ CT, Daily + (10문제/7문제)
        - 다이나믹 프로그래밍(), 애드 혹(), 이분탐색(집합과 맵), 최단경로(깊이우선탐색, 너비우선탐색), 비트마스킹(), 분리집합()
        - 트리를 사용한 집합과 맵(집합과 맵, 트리), 우선순위큐(스택, 큐), 분할정복(), 투포인터(), 유클리드호제법(), 격자그래프(), 슬라이딩윈도우(투포인터)
    - 세부알고리즘 ⇒ CT, Daily + (7문제/5문제)
        - 세그먼트트리(트리, 스택), 데이크스트라(최단경로), 최대유량(), 트라이(문자열), 최소스패닝트리(우선순위 큐)
        - 위상정렬(트리), 최소공통조상(트리), 플로이드로셜(최단경로)
        - 느리게 갱신되는 세그먼트 트리(세그먼트 트리)
        - 트리에서의 다이나믹 프로그래밍(다이나믹 프로그래밍, 트리)
        - 비트필드를 이용한 다이나믹 프로그래밍(다이나믹 프로그래밍, 비트마스킹)
        - 배낭문제(다이나믹 프로그래밍), 역추적(다이나믹 프로그래밍), 가장 긴 증가하는 부분수열 문제(다이나믹 프로그래밍)
    - 도전알고리즘 ⇒ Daily + (7문제/5문제)
        - 최소 비용 최대 유량(최대유량), KMP(문자열)
        - 외판원 순회(비트필드를 이용한 다이나믹 프로그래밍)
        - 이분매칭(최대유량), 강한연결요소(깊이우선탐색, 스택), 볼록 껍질(기하학)
        
** 나머지 태그는 제외된 태그로 표시(제외 사유는 기본값)
